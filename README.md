# GamePulse: Gaming Player Retention and CRM Analytics

## Stage 1: Dataset Setup and First Analysis

This project explores how a gaming company can use player behaviour data to understand retention, churn, user acquisition channels and CRM opportunities.

At this stage, the project uses a simulated gaming dataset designed for learning and portfolio development.

## Files in this starter pack

- `data/raw/gamepulse_players_stage1.csv`: simulated player dataset
- `data/raw/data_dictionary.md`: explanation of each column
- `notebooks/01_stage1_load_and_understand_data.ipynb`: first beginner-friendly notebook

## What Stage 1 covers

1. Loading the dataset
2. Checking rows and columns
3. Checking missing values
4. Understanding basic statistics
5. Calculating retention and churn
6. Reviewing acquisition channels
7. Exploring first simple player segments

## Tools

- Python
- pandas
- Jupyter Notebook or Google Colab

## Important note

This dataset is simulated for portfolio learning purposes. Later stages can include public gaming datasets and more advanced analysis.
