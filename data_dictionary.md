# GamePulse Stage 1 Data Dictionary

This is a simulated gaming dataset created for a beginner-friendly portfolio project.
It is designed to practise player behaviour analysis, retention analysis, CRM segmentation and user acquisition insight.

| Column | Meaning |
|---|---|
| player_id | Unique player identifier |
| install_date | Date the player installed the game |
| country | Player country |
| device_type | Device used by the player |
| acquisition_channel | Marketing/user acquisition source |
| campaign_name | Campaign linked to the acquisition channel |
| sessions_day_1 | Number of sessions on day 1 |
| sessions_day_7 | Number of sessions by day 7 |
| total_playtime_minutes | Total playtime in minutes |
| levels_completed | Number of levels completed |
| in_app_purchases_gbp | Total in-app purchases in GBP |
| ad_views | Number of ads viewed by the player |
| retained_day_1 | 1 if retained on day 1, otherwise 0 |
| retained_day_7 | 1 if retained on day 7, otherwise 0 |
| retained_day_30 | 1 if retained on day 30, otherwise 0 |
| churned | 1 if the player did not retain by day 30, otherwise 0 |
| basic_player_segment | Simple first-pass player segment for beginner analysis |

Important note:
This dataset is simulated for portfolio and learning purposes. In the README, you should state this clearly.
